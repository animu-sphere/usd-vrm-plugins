# OST v0.19.0 dogfooding — v0.3.0 release blocked by aggregate product reproducibility

- Date: 2026-07-23
- Version exercised: `ost 0.19.0` in GitHub Actions on Windows, Linux, and macOS arm64
- Release run: [Actions run #29948746190](https://github.com/animu-sphere/usd-vrm-plugins/actions/runs/29948746190)
- Related report: [26](26-2026-07-23-v0.19.0-v0.20.0-asks.md)
- Next ask target: `ost 0.20.0`

## TL;DR

The v0.3.0 release did not fail in a build, test, runtime, or operating-system
lane. All three release cells reached packaging successfully. They then failed
the new aggregate-product reproducibility check because two invocations of
`ost plugin package --workspace --product` produced different digests for the
aggregate product.

The four member package digests were stable in every cell. Only
`product:usd-vrm-plugins` changed. This isolates the failure to aggregate
product packaging in `ost 0.19.0`, not to the Windows, Linux, or macOS build
inputs.

## 1. What CI measured

The release workflow packages once, then repeats the exact command and compares
the reported archive digests:

```console
$ ost plugin package --workspace --product --target cy2026 --profile usd --json
```

The preflight, bootstrap, runtime verification, build, source-tree test, and
first packaging steps all passed in every OS cell. The only failed step was
`Prove packaging is digest-reproducible`.

Windows:

```diff
- product:usd-vrm-plugins sha256:543e349e0bb288b7dec107d6c765b7d8dbb57bd636ac60ad8af1a0cba58e3173
+ product:usd-vrm-plugins sha256:81dec8fcc8d0eb511b487b045ca9cf39940f0bbb60d684715096b11a919d655e
```

Linux:

```diff
- product:usd-vrm-plugins sha256:c372d8e462a1deb4930406176973720b77f813dc173201b718783eb2fd7c3186
+ product:usd-vrm-plugins sha256:49ee804593460d6a780fc6b86ee2588bf03e008b69361f15b7a66cdbda15181a
```

macOS arm64:

```diff
- product:usd-vrm-plugins sha256:6672232f47aeaff27b9713cd535b9df866300646acc89b57f64e88388aaabcd7
+ product:usd-vrm-plugins sha256:c1272e324e80c4763f01702b1e9e6dd0d3162e274161d4e5b20a76c21951f847
```

In all three diffs, the member archive lines immediately below the product
line were unchanged. The aggregate archive is therefore not a stable function
of an unchanged build, and release checksums cannot be trusted yet.

## 2. Why v0.3.0 exposed it

The v0.2.0 release lane used `ost plugin package --workspace` and compared only
the member package digests. v0.3.0 adopted the v0.19.0 aggregate product feature
and changed the lane to `--workspace --product`; it also added the product
digest to the reproducibility gate. The relevant workflow change is in commit
`4947cdb`.

This is why the three OS lanes appear to fail simultaneously: they are running
the same new product-level assertion, not independently hitting three
platform-specific failures.

The workflow log retains the JSON digest comparison but not both product archive
files, so this run cannot identify which byte or member inside the aggregate
archive changes. The evidence is sufficient to assign the defect to the
aggregate-product implementation, but not to claim a more specific internal
cause such as timestamps, ordering, or archive metadata.

## 3. Impact

The `assemble release (draft)` job was skipped because all build matrix cells
failed. No draft GitHub release or release assets were created. The member
packages may be individually reproducible, but the advertised aggregate
product and its checksum are not release-safe.

## 4. v0.20.0 ask

**Make `ost plugin package --workspace --product` byte-reproducible.**

Acceptance:

1. Build an unchanged workspace once.
2. Run `ost plugin package --workspace --product` at least twice without
   changing the build tree.
3. Require the aggregate product archive digest to match exactly, on Windows,
   Linux, and macOS.
4. Preserve deterministic member ordering, archive metadata, and product
   manifest contents so the resulting release checksum is stable.

Until this is fixed upstream, the release lane should keep the product digest
gate enabled. Removing that comparison would only hide an unstable artifact.
