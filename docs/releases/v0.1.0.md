# v0.1.0 — first public release

**Tagged:** 2026-07-12 (`c4ac304`) · **Published:** 2026-07-11 ·
[Release page](https://github.com/animu-sphere/usd-vrm-plugins/releases/tag/v0.1.0)

The first tagged release: a `.vrm` (VRM 0.x / 1.0) importer that normalizes into
a canonical model and authors a typed USD stage, plus the reliability tooling
around it.

## Objective

Ship an importer that is installable from a clean environment, opens a real
avatar, registers its schemas, passes its own validator, and produces
reproducible artifacts — proving the import/evaluation boundary end to end
before any runtime layer exists.

## What shipped

- **Importer** (`SdfFileFormat` for `vrm`): GLB read via cgltf, VRM 0.x / 1.0
  detection, all version differences absorbed into `VrmCanonicalDocument` before
  any USD is authored.
- **Geometry & materials:** `UsdGeomMesh`, non-skinned node transforms
  preserved, `UsdPreviewSurface` with the full texture set.
- **Skeleton & skinning:** one `UsdSkelSkeleton` unified across all glTF skins,
  bind from inverse bind matrices.
- **Skeletal animation:** glTF joint TRS clips → `UsdSkelAnimation`.
- **Front-direction bake:** VRM 0.x −Z front normalized to a canonical +Z rest
  pose.
- **Typed schemas** — `VrmHumanoidAPI`, `VrmExpressionAPI`, `VrmLookAtAPI`,
  `VrmSpringBoneAPI`, `VrmColliderAPI`, `VrmConstraintAPI` — authored as **data
  only**; evaluation and simulation were deferred to a future runtime layer.
- **MToon:** preserved as `vrm:mtoon:raw` + `vrm:shaderModel` tag, with a
  `UsdPreviewSurface` fallback.
- **Reliability tooling:** standalone stage validator, coded `VRMxxx` diagnostic
  taxonomy, compatibility report, portable texture packaging.
- **Release machinery:** digest-reproducible packaging, packaged-artifact
  verification, clean-install smoke on all three OS, `buildInfo.json` stamp.

Granular detail: [delivery history](../reports/delivery-history.md).

## Compatibility

- **Schema contract version:** 1
- **OpenUSD:** cy2026 / OpenUSD 26.05 (the tolerated range is `>=25.05,<27.0`;
  only cy2026 was verified in CI)
- **Platforms:** Windows x86_64, macOS arm64, Linux x86_64

## Artifacts

Released as `usdVrm-0.1.0-<target>.tar.zst` (+ `.manifest.json` sidecars), a
source archive, and `SHA256SUMS`.

> **These asset names are historical.** At v0.1.0 the importer was a single
> bundle named `usdVrm` with co-located schemas. Workspace Phase 4 retired
> `usdVrm` as a bundle id — it now names the aggregate product only — and the
> per-bundle artifact is `usdVrmFileFormat-<version>-<target>.tar.zst`. Anyone
> consuming the `usdVrm-*` asset names must update at the next release. See
> [WORKSPACE.md §5](../architecture/WORKSPACE.md).

## Known limitations at this release

- Morph-weight animation not authored.
- MToon is a `UsdPreviewSurface` approximation; no renderer-specific
  realization.
- One OpenUSD version cell only (no min-vs-latest axis).
- No runtime evaluation layer (`execVrm` did not exist).

## What changed after this release

Everything in the workspace split — `vrmSchema`, `vrmContainer`,
`usdVrmPackageResolver`, and the `usdVrm` → `usdVrmFileFormat` rename — landed
**after** v0.1.0 and is unreleased as of this writing. The next release is the
first to ship the multi-bundle workspace; see the
[roadmap](../roadmap/current.md).
