# v0.3.0 — the VRMA motion foundation

**Release status:** prepared on the v0.3.0 branch. The immutable tag and
published timestamp are recorded when the draft release is published.

## Objective

Ship the first motion boundary without smuggling a runtime into the importer:
accept a VRM Animation (`.vrma`) clip, author an avatar-independent semantic
USD skeleton and `UsdSkelAnimation`, and leave target binding and retargeting
for the next release.

## What ships

- **Motion Phase A:** the frozen
  [motion contract](../design/MOTION_CONTRACT.md), including source/target
  coordinate and time rules plus the hand-authored retarget hand-off triplet.
- **Workspace Phase 6a:** `motionCore`, a plain static CMake package of
  vendor-neutral humanoid values and declarative constraints. It exposes no
  USD stage or plugin API.
- **Workspace Phase 7 / Motion Phase B:** `usdVrmaFileFormat`, a VRMC VRM
  Animation 1.0 reader that maps humanoid rotations and hips translation from
  the first GLB animation to `/Animation/HumanoidSkeleton` and
  `/Animation/BodyAnimation`.
- **Hermetic source build:** cgltf v1.15 source and its MIT license are vendored
  under `third_party/cgltf`; no configure-time network fetch is required.
- **Release assurance:** a deterministic GLB fixture passes OST discovery,
  `usdcat`, Python stage-open, flattened-golden checks (L0–L5), and a
  value-level CTest. The VRMA package is also tested independently from its
  packaged artifact.

## Compatibility and artifacts

- **OpenUSD:** cy2026 / 26.05; tolerated range `>=25.05,<27.0`.
- **Platforms:** Windows x86_64, macOS arm64, Linux x86_64.
- **Schema contract:** 1, unchanged. VRMA v0.3.0 introduces no new typed VRM
  schema.

Per target the release publishes four member archives:
`vrmSchema-0.3.0-<target>.tar.zst`,
`usdVrmFileFormat-0.3.0-<target>.tar.zst`,
`usdVrmPackageResolver-0.3.0-<target>.tar.zst`, and
`usdVrmaFileFormat-0.3.0-<target>.tar.zst`, each with a manifest sidecar, plus
the aggregate product archive, source archive, and checksums.

The three VRM bundles remain one composed install. `usdVrmaFileFormat` is an
independent file-format package; it requires only the static `motionCore` and
the runtime library closure that OpenStrata packages with it.

## Known limitations

- A `.vrma` does not target an avatar in v0.3.0. Offline retargeting is Motion
  Phase C (`vrmRetarget`).
- Expressions, look-at, live capture, generation, OpenExec execution, blending,
  IK, and contact handling are out of scope.
- CUBICSPLINE tangents are not preserved; input values are read as a linear
  approximation with a warning. Scale and non-hips translation channels are
  ignored with a warning.
